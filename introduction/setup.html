Setup Instructions

  1. Install the Ebiz Airline Bot package to your Salesforce org.
    Step-by-step
  2. Create a Salesforce connected app for API.
    Step-by-step
  3. Create WIT AI account for bot conversation
    Step-by-step
  4. Create Account on http://www.air-port-codes.com for Airport API
    Step-by-step
  5. Google Places API
    Step-by-step
  6. Deploy a copy of the app to Heroku.
    Step-by-step
  7. Create Facebook AppStep-by-step
  8. Connect the Heroku Connect add-on to your Salesforce org, and import the configuration file.
    Step-by-step

Salesforce

  1. If you do not have a Salesforce org to connect Ebiz Airlines Bot create a free Developer Edition org.
  2. Install the Ebiz Airline Bot package by clicking the link below. If you have multiple Salesforce orgs, make sure you pick the one you would like to connect to your Ebiz Airline Bot app.

  3. Please note that, the above Sales install button supports for Salesforce free version. But, it does not support Apex Classes or Process Builder
    If you have Salesforce Professional Edition, please click the bellow button for install Ebiz Airlines Package that support Apex Classes and Process Builder

  4. Unless you have a reason otherwise, leave the default option (Install for Admins Only) selected when installing.

  5. Now you can check for Ebiz Airlines Bot in Installed App.

Salesforce Connected App

  1. In your Salesforce setup page, find App in Quick Find box. Click App Manager.
  2. In App Manager page, on the top right, click New Connected App

  3. Fill required information

    Please make sure you checked at Enable OAuth Settings

    Callback URL is: https://yourappname.herokuapps.com/oauth/_callback

    Make sure you selected at Access and manage your data (api) and Provide access to your data via the Web(web) as your app scopes

  4. Click Save. In next screen, at API (Enable OAuth Settings) section, get your Consumer Key (CLIENT_ID), Consumer Secret (CLIENT_SECRET) to prepare for Heroku config

Wit.AI

  1. Download WIT.AI App by click on this button

  2. Login to WIT.AI and click on the following link to create a new WIT.AI app https://wit.ai/apps/new

  3. Enter App Name, App Description and choose English as default language.
  4. Import your app from the backup file you downloaded at step 1 and click Create App
  5. After created app, click Setting and copy Server Access Token. Save it to somewhere, it will be the WIT_TOKEN in Heroku Config in few next steps

Airport Code API (free 14 days)

  1. Create an account at Air-port-codes at the following link https://www.air-port-codes.com/auth/register/
  2. Login with your new account
  3. Access the following link https://www.air-port-codes.com/auth/api-access/
  4. Click Request New API Secret to obtain API Secret(APC_AUTH_SECRET) and API Key (APC_AUTH) that required for Heroku config

Google Places API

  1. Go to https://console.developers.google.com/

  2. Create a new project

  3. Click Library on the left menu and select Google Places API Web Service in the Google Map API section

  4. Click Enable button to enable Google Map API

  5. On the left menu, click Credential button

  6. Select Create credentials button and select API key option

  7. Copy the API key. It will be used in the next step as GOOGLE_API

Heroku

  1. Deploy your own instance of Ebiz Airline Bot on Heroku by clicking the button below.

  2. Choose a unique name for your Ebiz Airline Bot instance.

  3. Define your config variables with information get from steps above except MESSENGER_PAGE_ACCESS_TOKEN and MESSENGER_APP_SECRET

  4. Please note that, MESSENGER_VALIDATION_TOKEN is anything you want. It will be required to verify your heroku URL in Facebook Messenger App in the next step.

Create Facebook App

  1. Create or configure a Facebook App or Page here https://developers.facebook.com/apps

  2. In the app go to Messenger tab then click Setup Webhook. Here you will put in the URL of your Heroku server and a token (Token is the string you defined in the previous step. You can get from Heroku config will the name is MESSENGER_VALIDATION_TOKEN ). Make sure to check all the subscription fields.

  3. Get a Page Access Token, Facebook App Secret and save to Heroku Config as MESSENGER_APP_SECRET and MESSENGER_PAGE_ACCESS_TOKEN.

  4. Go back to Terminal and type in this command to trigger the Facebook app to send messages. Remember to use the token you requested earlier. 

    curl -X POST "https://graph.facebook.com/v2.6/me/subscribed_apps?access_token=<MESSENGER_PAGE_ACCESS_TOKEN>"

Heroku Connect

  1. Click the Heroku Connect link in the Installed add-ons section of the app dashboard.
  2. Click the Setup Connection button.
  3. Select the HEROKU_CONNECT_DB Postgres database. Choose DATABASE_URL for DB Config Vars
  4. Click the Next button in the top right of the page.
  5. Click the Authorize button in the top right of the page.
  6. You will be prompted to Choose a Username. If you have multiple options, be sure to select the username associated with the Salesforce org your Ebiz Airline Bot instance should connect to
  7. You will be prompted to Allow Access?. Click the Allow button at the bottom right
  8. Now back to Heroku Connect Dashboard, choose Settings > Import/Export Configuration > click Import for mapping data with Salesforce

  9. Download the Heroku Connect configuration file to your computer.

  10. On the Heroku Connect page, click Choose a File and select the file you have downloaded
  11. Click the Upload button in the top right of the page

  12. The connection between your Ebiz Airline Bot database and Salesforce is now being set up. Keep an eye on the Mappings section. Wait until you have six objects with a status of OK before you continue. This may take a few minutes.
